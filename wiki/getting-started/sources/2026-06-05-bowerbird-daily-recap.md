A knowledge base you never revisit is a write-only archive; a daily recap fixes that.

- One message a day summarizing what was added, grouped by where it came from.
- Compute "what's new" from the wiki's git history, not post dates — otherwise a compiler catching up on a backlog makes old posts masquerade as today's news.
