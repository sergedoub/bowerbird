A knowledge base you never revisit is a write-only archive. The fix is a daily recap: one message summarizing what was added in the last day, grouped by where it came from.

Compute "what's new" from the wiki's git history, not from post dates — when the compiler catches up on a backlog, old posts would otherwise masquerade as today's news.
